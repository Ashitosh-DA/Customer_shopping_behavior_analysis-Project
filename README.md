# Customer Shopping Behavior Analysis

## Project Overview

This project analyzes customer shopping behavior using transactional retail data to uncover patterns in customer spending, product preferences, shopping frequency, subscription behavior, and discount usage. The objective is to transform raw shopping data into meaningful business insights that can support decision-making in marketing, customer retention, and product strategy.

The project combines **Python**, **PostgreSQL**, and **Power BI** to create an end-to-end analytics workflow:

* **Python** for data cleaning, preprocessing, and exploratory data analysis
* **PostgreSQL** for solving business-driven analytical questions using SQL
* **Power BI** for building an interactive dashboard to visualize insights

---

## Objective

The main goal of this project is to analyze customer purchase behavior and identify key business insights such as:

* Which customer groups generate the highest revenue
* How subscription status impacts customer spending
* Which products and categories perform best
* How discounts influence purchasing behavior
* Which customer segments are most valuable for retention and loyalty programs

This project demonstrates a complete **data analytics workflow** from raw data preparation to dashboard reporting.

---

## Dataset Summary

The dataset contains **3,900 customer purchase records** with **18 columns**, covering customer demographics, purchase information, and shopping behavior.

### Key Features

* **Customer Demographics**

  * Age
  * Gender
  * Location
  * Subscription Status

* **Purchase Information**

  * Item Purchased
  * Category
  * Purchase Amount
  * Season
  * Size
  * Color

* **Shopping Behavior**

  * Discount Applied
  * Promo Code Used
  * Previous Purchases
  * Frequency of Purchases
  * Review Rating
  * Shipping Type

### Data Quality

* The `Review Rating` column contained **37 missing values**
* Missing values were handled during data cleaning using **median imputation by product category**

## Project Workflow

### 1. Data Cleaning & Preparation in Python

The dataset was first cleaned and transformed using Python.

#### Steps performed:

* Loaded the dataset using **Pandas**
* Explored structure using `df.info()` and summary statistics using `df.describe()`
* Checked for null values across all columns
* Filled missing values in the **Review Rating** column using the **median rating within each product category**
* Standardized column names into **snake_case**
* Performed feature engineering:

  * Created **age_group** by grouping customers into age brackets
  * Created **purchase_frequency_days** from purchase-related information
* Checked redundancy between `discount_applied` and `promo_code_used`
* Dropped unnecessary/redundant columns where applicable
* Exported the cleaned dataset to **PostgreSQL** for SQL-based business analysis

## SQL Business Questions Solved

After cleaning the data, the processed dataset was loaded into PostgreSQL to answer business-focused analytical questions.

### 1. Revenue by Gender

Compared the total revenue generated by male and female customers.

### 2. High-Spending Discount Users

Identified customers who used discounts but still spent above the average purchase amount.

### 3. Top 5 Products by Rating

Found the products with the highest average review ratings.

### 4. Shipping Type Comparison

Compared average purchase amounts across different shipping methods such as standard and express shipping.

### 5. Subscribers vs Non-Subscribers

Compared total revenue and average spending between subscribed and non-subscribed customers.

### 6. Discount-Dependent Products

Identified the top products with the highest percentage of discounted purchases.

### 7. Customer Segmentation

Segmented customers into:

* **New Customers**
* **Returning Customers**
* **Loyal Customers**

Segmentation was based on purchase history and repeat purchase behavior.

### 8. Top 3 Products per Category

Ranked the most purchased products within each product category.

### 9. Repeat Buyers & Subscription Behavior

Checked whether customers with more than 5 purchases are more likely to subscribe.

### 10. Revenue by Age Group

Calculated total revenue contribution by different age groups to identify high-value customer segments.

## Key Insights

Some of the major insights generated from the analysis include:

* Certain customer groups contribute significantly more to total revenue than others
* Subscription status has a noticeable impact on overall customer spending
* Some products are highly dependent on discounts for sales volume
* Repeat buyers show strong potential for loyalty and subscription-based marketing
* Specific age groups contribute more revenue and can be targeted more effectively
* Top-rated and top-selling products can be prioritized in promotions and campaigns
* Shipping preference may also influence customer spending behavior
